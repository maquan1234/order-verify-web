# 企业微信「实时核查机器人」配置说明

> 目标：同事在企微群里发一张销售单 Excel，机器人自动核对，把结果发回群里。
> 代码已写好（`wecom_bot.py` + `app.py` 的 `/wecom/callback`）。你只需按下面 4 步把凭证填好。

---

## 0. 前置硬条件（必须，否则回调收不到）

企业微信要求回调地址是 **固定不变的公网 https 地址**，且要验证域名所有权。

- ❌ 不可用：`启动并联网.bat` 生成的 Cloudflare 临时隧道（每次重启域名都变）。
- ✅ 可用（任选其一）：
  - **云部署**：把 GitHub 仓库 `maquan1234/order-verify-web` 部署到 Render / Railway / Koyeb 等，拿到永久域名（需你注册一个账号，一次性）。
  - **Cloudflare 保留隧道（付费很便宜）**：在你电脑上用 `cloudflared tunnel` 配置「保留隧道」，拿到固定子域名。
- 拿到固定地址后，它必须支持 HTTPS（上面两种都满足）。

下文把这个固定地址记作 `https://你的域名`。

---

## 1. 注册企业微信并创建自建应用

1. 打开 https://work.weixin.qq.com ，用微信扫码注册（免费，个人/个体户都能注册）。
2. 登录后「我的企业 → 企业信息」记下 **企业 ID（corpid）**。
3. 「应用管理 → 自建 → 创建应用」，名字随意（如“审单核查”），可见范围选要用的人。
4. 进入该应用，记下 **AgentId** 和 **Secret**（Secret 要点“查看”用微信扫码获取）。

---

## 2. 设置「接收消息」回调

1. 在应用详情页拉到最下面，点 **接收消息 → 设置 API 接收**。
2. 填：
   - **URL**：`https://你的域名/wecom/callback`
   - **Token**：自己随便写一段（如 `shendan123`），记住它。
   - **EncodingAESKey**：点“随机生成”，记下这串 43 字符。
3. 点「保存」——此时企业微信会立即向你填的 URL 发一次验证请求。
   - 你的服务必须**已经在跑且公网可达**（见第 4 步），否则保存失败。
   - 验证通过后，应用就能收到群里发的消息了。

> 若保存时提示域名未验证：去「我的企业 → 可信域名」添加 `你的域名`，
> 下载校验文件 `WW_verify_xxxx.txt`，把它的**文件名**和**内容**分别填到下面环境变量
> `WECOM_VERIFY_FILENAME` / `WECOM_VERIFY_CONTENT`，重启服务后再保存即可。

---

## 3. 填好环境变量（凭证）

在你运行服务的方式里注入以下环境变量（示例用 Windows 的 `set`；云部署则在平台的环境变量面板填）：

```
set WECOM_ENABLED=1
set WECOM_CORPID=你的企业ID
set WECOM_SECRET=你的应用Secret
set WECOM_AGENTID=你的AgentId
set WECOM_TOKEN=第2步填的Token
set WECOM_AES_KEY=第2步生成的EncodingAESKey
set PUBLIC_BASE_URL=https://你的域名
set WECOM_VERIFY_FILENAME=WW_verify_xxxx.txt   （仅域名验证阶段需要）
set WECOM_VERIFY_CONTENT=校验文件内容            （仅域名验证阶段需要）
```

不会改 bat？把上面的 `set` 行加进 `启动并联网.bat` 里 `cloudflared` 之前即可（部署到云则在平台控制台填）。

---

## 4. 启动 & 把应用拉进群

- 本地/服务器启动服务（确保 `WECOM_ENABLED=1` 已生效）。
- 在企业微信里，**把这个自建应用拉进你要用来核查的群**（建议专门建一个“审单核查群”）。
- 群里任何人发一张 `.xlsx` 销售单 → 机器人收到 → 几秒后把核查结果（订单数 / 异常数 / 结果下载链接）发回群里。

---

## 5. 自测加解密（可选，上线前确认）

```bash
pip install cryptography
python wecom_bot.py <Token> <EncodingAESKey> <corpid>
```
无报错即表示加解密实现正确。

## 常见问题

- **保存 API 接收时一直转圈/失败**：服务没公网可达，或 `/wecom/callback` 没返回明文 echostr。先看服务日志。
- **群里发文件机器人没反应**：确认应用已被拉进该群、且 `WECOM_ENABLED=1`；看服务控制台有无 `[wecom]` 报错。
- **回消息报没权限**：本实现用 `appchat/send`，适用于“应用自建群”。若你用的是普通内部群，
  可能需要改用 `message/send` 回给具体成员——把你的群类型告诉我，我再调发送接口。
- **结果下载链接打不开**：确认 `PUBLIC_BASE_URL` 填的是同一个公网地址。

> 注意：企业微信回调对稳定性要求较高，建议用云部署（7×24 在线），比“自己电脑开着”更稳。
