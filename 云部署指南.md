# 云部署指南（拿到固定公网域名，供企微回调）

目标：把 `github.com/maquan1234/order-verify-web` 部署到云平台，拿到一个**固定不变的公网 https 地址**（如 `https://order-verify-web.onrender.com`），企业微信才能把群里的表格推过来。

## 方式一：Render（推荐，免费）
1. 打开 https://render.com → 右上角 **Sign Up** → 选 **GitHub** 登录授权（之前转圈就换个网络/浏览器再试，或改用方式二）。
2. 登录后点 **New + → Web Service** → 选 **Build and deploy from a Git repository** → 关联你的仓库 `maquan1234/order-verify-web`。
3. 配置：
   - Runtime: **Python 3**
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python app.py`
   - Plan: **Free**
4. 点 **Advanced → Add Environment Variable**，把下面 7 个变量逐个粘贴（值见你本机的 `.env`）：
   - `WECOM_ENABLED` = `1`
   - `WECOM_CORPID` = `ww97d75368178a37ee`
   - `WECOM_SECRET` = 你的 Secret
   - `WECOM_AGENTID` = `1000002`
   - `WECOM_TOKEN` = 你的 Token
   - `WECOM_AES_KEY` = 你的 EncodingAESKey
   - `PUBLIC_BASE_URL` = 部署完成后的域名（先留空，第 5 步拿到再回来填）
5. 点 **Create Web Service**，等 1–2 分钟构建完。页面顶部会显示一个域名，形如 `https://order-verify-web-xxxx.onrender.com`。
6. 回到 Render 后台，把 `PUBLIC_BASE_URL` 补填为这个域名（含 `https://`）。
7. 回到企业微信后台「设置 API 接收」：URL 填 `https://你的域名/wecom/callback`，Token / EncodingAESKey 照填，保存即可（此时服务已在跑，验证会通过）。

## 方式二：Koyeb（GitHub 登录若 Render 不通可试）
1. 打开 https://app.koyeb.com → 用 GitHub 登录。
2. **Create App → Create a web service** → 关联 GitHub 仓库 → 选 `Dockerfile` 或 `Build from GitHub`（平台会自动识别 Python）。
3. 启动命令 `python app.py`，在 **Environment variables** 里填同样的 7 个变量。
4. 部署后拿到 `*.koyeb.app` 域名，同样回填 `PUBLIC_BASE_URL` 并完成企微「设置 API 接收」。

## 注意事项
- 免费实例**闲置约 15 分钟后会休眠**，下次访问冷启动约 30–50 秒；群里发文件时若服务在睡，第一条可能慢，属正常。
- 云端 `products.json` 由仓库种子提供；在网页里改货品**不会永久保存**（重启即还原），需要持久改货品就改仓库里的 `products.json` 再重新部署。
- 免费额度够个人/小团队用；量大再升级付费套餐。
